# 基于协同过滤的技能交易系统 — 启动文档

## 项目结构

```
jinengjiaoyi/
├── jinengjiaoyi-behind/    后端服务（Spring Boot）    端口 48081
├── jinengjiaoyi-user/      用户端前端（Vue 3）        端口 8081
├── jinengjiaoyi-admin/     管理端前端（Vue 3）        端口 8082
└── db/                     数据库脚本
```

## 一、数据库配置

1. 创建数据库：

```sql
CREATE DATABASE jinengjiaoyi DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

1. 导入数据：

```bash
mysql -u root -p jinengjiaoyi < jinengjiaoyi-behind/db/jinengjiaoyi.sql
```

1. 数据库连接配置在 `jinengjiaoyi-behind/src/main/resources/application.yml` 中，默认配置如下：


| 配置项  | 默认值            |
| ---- | -------------- |
| 地址   | 127.0.0.1:3306 |
| 数据库名 | jinengjiaoyi   |
| 用户名  | root           |
| 密码   | root123        |


如需修改，编辑 `application.yml` 中对应字段即可。

## 二、启动后端服务

```bash
cd jinengjiaoyi-behind
mvn spring-boot:run
```

启动成功后，后端 API 运行在 **[http://localhost:48081](http://localhost:48081)**

验证后端是否启动成功：

```bash
curl http://localhost:48081
```

## 三、启动用户端前端

```bash
cd jinengjiaoyi-user
npm install
npm run dev
```

启动成功后，用户端运行在 **[http://localhost:8081](http://localhost:8081)**

浏览器访问 [http://localhost:8081](http://localhost:8081) 即可进入用户端登录页面。

## 四、启动管理端前端

```bash
cd jinengjiaoyi-admin
npm install
npm run dev
```

启动成功后，管理端运行在 **[http://localhost:8082](http://localhost:8082)**

浏览器访问 [http://localhost:8082](http://localhost:8082) 即可进入管理端登录页面。

## 七、生产部署

### 后端打包

```bash
cd jinengjiaoyi-behind
mvn clean package -DskipTests
java -jar target/jinengjiaoyi-0.0.1-SNAPSHOT.jar
```

### 前端打包

```bash
# 用户端
cd jinengjiaoyi-user
npm run build
# 产物在 dist/ 目录

# 管理端
cd jinengjiaoyi-admin
npm run build
# 产物在 dist/ 目录
```

### Nginx 配置参考（前端统一部署到 8081 端口）

```nginx
server {
    listen 8081;
    server_name localhost;

    # 用户端
    location / {
        root /path/to/jinengjiaoyi-user/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 管理端
    location /admin {
        alias /path/to/jinengjiaoyi-admin/dist;
        index index.html;
        try_files $uri $uri/ /admin/index.html;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://localhost:48081/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://localhost:48081/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 八、常见问题

**Q: 前端页面打开后接口报错 CORS？**
A: 确保后端已启动且 `InterceptorConfig.java` 中包含 CORS 配置。

**Q: 登录后跳转空白？**
A: 检查前端 `config.js`（用户端）或 `base.js`（管理端）中的后端地址是否正确指向 `http://localhost:48081/`。

**Q: npm install 失败？**
A: 尝试使用国内镜像：`npm install --registry=https://registry.npmmirror.com`

**Q: Maven 编译失败？**
A: 确保使用 JDK 1.8，检查 `JAVA_HOME` 环境变量配置。
