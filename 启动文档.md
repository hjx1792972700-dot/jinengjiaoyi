# 基于协同过滤的技能交易系统 — 启动文档

## 项目结构

```
jinengjiaoyi/
├── jinengjiaoyi-behind/    后端服务（Spring Boot）       端口 48081
├── jinengjiaoyi-user/      用户端前端（Vue 2 + Element UI） 端口 8081
├── jinengjiaoyi-admin/     管理端前端（Vue 2 + Element UI） 端口 8082
```

## 一、数据库配置

1. 创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS jinengjiaoyi DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

1. 数据库连接配置在 `jinengjiaoyi-behind/src/main/resources/application.yml` 中，默认配置如下：


| 配置项  | 默认值            |
| ---- | -------------- |
| 地址   | 127.0.0.1:3306 |
| 数据库名 | jinengjiaoyi   |
| 用户名  | root           |
| 密码   | root123        |


如需修改，编辑 `application.yml` 中对应字段即可。

> **注意**：本项目使用 **Liquibase** 管理数据库版本，**无需手动导入 SQL 文件**。后端首次启动时，Liquibase 会自动执行 `resources/db/changelog/` 下的 15 个 changelog 文件（共 160 个 changeSet），自动创建所有表结构并导入测试数据。

## 二、启动后端服务

```bash
cd jinengjiaoyi-behind
mvn spring-boot:run -DskipTests
```

启动成功后，控制台会显示 `Tomcat started on port(s): 48081`，后端 API 运行在 **[http://localhost:48081](http://localhost:48081)**

验证后端是否启动成功：

```bash
curl http://localhost:48081
# 应重定向到 index.html
```

## 三、启动用户端前端

```bash
cd jinengjiaoyi-user
npm install
npm run dev
```

启动成功后，用户端运行在 **[http://localhost:8081](http://localhost:8081)**

浏览器访问 [http://localhost:8081](http://localhost:8081) 即可进入用户端登录页面。

## 四、启动管理端前端

```bash
cd jinengjiaoyi-admin
npm install
npm run dev
```

启动成功后，管理端运行在 **[http://localhost:8082](http://localhost:8082)**

浏览器访问 [http://localhost:8082](http://localhost:8082) 即可进入管理端登录页面。

## 七、生产部署

### 方式一：嵌入式部署（推荐）

前端构建后嵌入后端，只需运行一个 Spring Boot 服务即可：

```bash
# 1. 构建用户端
cd jinengjiaoyi-user
npm run build
# 复制 dist 到后端
cp -r dist/ ../jinengjiaoyi-behind/src/main/resources/front/dist/

# 2. 构建管理端
cd ../jinengjiaoyi-admin
npm run build
# 复制 dist 到后端
cp -r dist/ ../jinengjiaoyi-behind/src/main/resources/admin/dist/

# 3. 打包并运行后端
cd ../jinengjiaoyi-behind
mvn clean package -DskipTests
java -jar target/jinengjiaoyi-0.0.1-SNAPSHOT.jar
```

访问方式：

- 用户端：[http://localhost:48081/front/index.html](http://localhost:48081/front/index.html)
- 管理端：[http://localhost:48081/admin/index.html](http://localhost:48081/admin/index.html)

### 方式二：独立部署（Nginx 反向代理）

```nginx
server {
    listen 80;
    server_name localhost;

    # 用户端
    location / {
        root /path/to/jinengjiaoyi-user/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 管理端
    location /admin {
        alias /path/to/jinengjiaoyi-admin/dist;
        index index.html;
        try_files $uri $uri/ /admin/index.html;
    }

    # 后端 API 反向代理
    location /jinengjiaoyi/ {
        proxy_pass http://localhost:48081/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://localhost:48081/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 八、常见问题

**Q: 前端页面打开后接口报错 CORS？**
A: 确保后端已启动。后端 `InterceptorConfig.java` 中已配置全局 CORS 策略。

**Q: 登录后跳转空白？**
A: 检查前端 `config.js`（用户端）或 `base.js`（管理端）中的后端地址是否指向 `http://localhost:48081/`。

**Q: npm install 失败？**
A: 使用国内镜像：`npm install --registry=https://registry.npmmirror.com`

**Q: Maven 编译失败？**
A: 确保使用 JDK 1.8，检查 `JAVA_HOME` 环境变量。

**Q: 数据库表没有自动创建？**
A: 检查 `application.yml` 中 `spring.liquibase.enabled` 是否为 `true`（默认开启）。确保 MySQL 数据库 `jinengjiaoyi` 已手动创建且连接配置正确。

**Q: 启动报 Liquibase 错误？**
A: 通常是数据库连接信息不正确，或数据库不存在。确认 MySQL 服务已启动、数据库已创建、用户名密码正确。
