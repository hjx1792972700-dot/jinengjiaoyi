# 基于协同过滤的技能交易系统 — 毕业设计答辩讲解文档

## 一、选题背景与研究意义

### 1.1 背景

在大学校园中，学生个体之间存在显著的技能差异——有人擅长编程，有人精于设计，有人掌握乐器，有人熟悉外语。然而，传统的技能获取方式（如报培训班、自学）存在成本高、效率低的问题。与此同时，许多拥有特长的学生缺乏合适的渠道分享和交换自己的技能。

本系统旨在构建一个**校园技能交易平台**，让学生可以发布自己的技能供给和学习需求，通过**技能交换**的方式实现互助学习。系统引入**协同过滤推荐算法**，根据用户行为（收藏、浏览、交换历史）智能推荐匹配的技能和用户，提高交换效率。

### 1.2 研究意义

- **实际意义**：解决校园内技能信息不对称问题，降低技能获取成本，促进学生间互助交流
- **技术意义**：将协同过滤推荐算法应用于技能匹配场景，探索用户-物品矩阵在非电商领域的实践
- **社会意义**：推动共享经济理念在教育领域的应用，促进校园知识共享生态的建设

## 四、核心功能模块详解

### 4.1 用户认证与权限管理

**实现方式**：自定义 Token 认证拦截器（非 Spring Security / Shiro）

**认证流程**：

1. 用户登录（`/yonghu/login` 或 `/users/login`）→ 服务端生成 32 位随机 Token，存入 `token` 表，有效期 1 小时
2. 前端将 Token 存入 `localStorage`，后续请求通过 HTTP Header `Token` 携带
3. `AuthorizationInterceptor` 拦截所有 API 请求，校验 Token 有效性和过期时间
4. 公开接口通过 `@IgnoreAuth` 注解跳过认证（如登录、注册、技能列表浏览）

**角色区分**：

- **管理员**（`users` 表）：访问管理后台，进行审核、统计、系统配置
- **普通用户**（`yonghu` 表）：发布技能/需求、交换、社区互动
- 两套角色使用独立的登录入口和 Token 存储键

**代码位置**：`com.interceptor.AuthorizationInterceptor`、`com.service.impl.TokenServiceImpl`

### 4.2 技能市场（技能发布与需求发布）

**数据模型**：`jinengxuqiu` 表通过 `leixing` 字段区分"技能"和"需求"

**发布流程**：

1. 用户填写标题、分类、描述、详情、费用、封面图片
2. **VIP 权限校验**：发布"技能"类型要求用户具备 VIP 身份
3. 提交后进入**审核队列**，`sfsh` 字段初始值为"待审核"
4. 管理员审核通过（`sfsh='是'`）后，技能/需求在市场中可见

**审核状态机**：

```
待审核 ──管理员通过──→ 是（已通过，前台可见）
  │
  └──管理员拒绝──→ 否（已拒绝，附拒绝理由 shhf）
```

**交互功能**：每条技能/需求支持点赞、收藏、评论、浏览计数，这些行为数据同时作为推荐算法的输入特征。

**代码位置**：`com.controller.JinengxuqiuController`

### 4.3 技能交换工作流（核心业务）

这是系统最核心的业务模块，实现了完整的**交换申请 → 选择技能 → 确认交换 → 生成记录**的闭环流程。

**完整状态流转**：

```
用户A发布需求 ←────────────────────────────────────────────────┐
     │                                                        │
用户B浏览需求，发起交换申请                                       │
     │                                                        │
     ▼                                                        │
┌──────────┐   用户A选择B的某项技能    ┌──────────┐            │
│  待审核   │ ───────────────────→  │  待确认   │            │
└──────────┘                       └──────────┘            │
     │                                  │                    │
     │                         用户B确认交换                  │
     │                                  │                    │
     │                                  ▼                    │
     │                          ┌──────────┐                 │
     │                          │  已完成   │ → 自动生成交换记录│
     │                          └──────────┘                 │
     │                                                        │
     └───── 拒绝 ───→ 否（附拒绝理由）──────────────────────────┘
```

**关键实现逻辑**（`JiaohuanshenqingController`）：

1. **申请阶段** (`add`)：
  - 防重复申请检查（`checkDuplicate`）
  - 禁止自我申请
  - 创建申请记录，状态为"待审核"
2. **选择技能阶段** (`selectSkill`)：
  - 需求发布者查看收到的申请列表（`receivedApply`）
  - 选择申请人提供的某项技能，记录 `xuanzejinengid` 和 `xuanzejinengbiaoti`
  - 状态从"待审核"变为"待确认"
3. **确认交换阶段** (`confirmExchange`)：
  - 申请人确认交换
  - 状态从"待确认"变为"已完成"
  - **自动生成** `jiaohuanjilu`（交换记录），快照双方信息和提供技能
4. **拒绝** (`rejectExchange`)：状态变为"否"，附拒绝理由

### 4.4 推荐算法模块

#### 4.4.1 算法架构

系统实现了**多层次推荐策略**，包含基于用户的协同过滤、基于物品的协同过滤和神经协同过滤（NCF）。

**推荐算法工厂**（`RecommendAlgorithmFactory`，约 1400 行）：

```
推荐请求
    │
    ▼
┌─────────────────────────────────────────────┐
│          RecommendAlgorithmFactory            │
│                                              │
│  1. 构建用户-物品行为矩阵                       │
│     （数据源：storeup 收藏表）                   │
│                                              │
│  2. 计算用户相似度 / 物品相似度                   │
│     （余弦相似度）                              │
│                                              │
│  3. 协同过滤推荐                               │
│     ├── User-Based CF（基于相似用户的偏好）       │
│     └── Item-Based CF（基于相似物品的关联）       │
│                                              │
│  4. NCF 深度学习模型（可选）                     │
│     ├── 训练：Weka RandomForest/MLP            │
│     ├── 模型持久化：.model 文件                  │
│     └── 预测：加载模型推荐物品                    │
│                                              │
│  5. 混合策略 + 热门回退                         │
└─────────────────────────────────────────────┘
```

#### 4.4.2 技能推荐实现 (`autoSort2`)

```java
// 核心推荐流程（JinengxuqiuController.autoSort2）

// Step 1: 提取当前用户的需求向量
//   - 获取用户所有 leixing='需求' 的记录
//   - 按 jinengfenlei 构建需求分类向量

// Step 2: 计算用户间相似度
//   - 遍历所有用户的需求向量
//   - 使用余弦相似度计算相似程度

// Step 3: 从相似用户的收藏记录中提取推荐候选
//   - 获取最相似用户群体的 storeup 记录（type=1 或 type=21）
//   - 过滤掉当前用户已浏览/已收藏的物品

// Step 4: 加载训练好的 NCF 模型补充推荐
//   - RecommendAlgorithmFactory.loadModel("jinengxuqiu.model")
//   - 调用 recommendItems 获取模型预测结果

// Step 5: 合并排序，不足时用热门列表回退
//   - ORDER BY FIELD(id, 推荐ID列表)
```

#### 4.4.3 用户推荐实现 (`/yonghu/recommend`)

根据当前用户的需求分类，推荐拥有匹配技能的用户，综合**技能匹配度**和**信誉指数**进行排序。

#### 4.4.4 模型训练 (`trainRecommend`)

- 从 `yonghu`、`jinengxuqiu`、`storeup` 三表构建训练特征
- 使用 Weka 的 RandomForest 或 MultilayerPerceptron 进行 NCF 训练
- 训练结果序列化保存到 `algorithm/recommend/models/` 目录

### 4.5 VIP 会员系统

**会员等级**：月度会员、季度会员、年度会员、连续包月

**会员特权**：

- 发布"技能"类型（非 VIP 用户只能发布"需求"）
- 免费访问付费学习资料（`checkPurchased` 中 VIP 用户直接放行）

**充值流程**（`HuiyuanchongzhiController`）：

1. `createOrder`：创建充值订单
2. `payCallback`：支付回调，更新订单状态为"已支付"，更新用户 VIP 状态和到期时间
3. `vipStatus`：查询当前 VIP 状态，自动处理过期降级

### 4.6 学习资料模块

- 资料分为 8 个类型（产品经理、软件研发、新媒体运营等）
- 每份资料包含封面、视频、附件、详情、价格
- 支持免费和付费资料，付费资料通过 `ziliaogoumai` 订单流程购买
- VIP 用户可免费获取所有付费资料
- 集成 **DPlayer** 视频播放器支持在线观看

### 4.7 社区论坛

- 7 个论坛分类，支持发帖、回帖（`parentid` 树形结构）
- 帖子审核机制（管理员审核通过后可见）
- 支持匿名发帖、置顶帖、点赞、收藏
- 管理员可批量审核帖子

### 4.8 即时通信

**双通道消息系统**：

1. **用户私信**（`chatmessage` + WebSocket）：
  - `WebSocketManager` 使用 `@ServerEndpoint("/ws")` 实现
  - 支持 `user_id` / `to_id` 参数建立点对点连接
  - 消息实时推送，支持已读/未读状态标记
  - 内存维护连接映射（`ConcurrentHashMap`），支持 ping/pong 心跳
2. **AI 智能问答**（`chat` + CommonController）：
  - 通过 `/askai` 和 `/askai/stream`（SSE 流式输出）调用 AI 模型
  - 历史对话持久化到 `chat` 表
  - 预设常见问答库（`chathelper`）

## 五、系统安全设计

### 5.1 认证机制

- **Token 拦截器**：所有 API 请求经过 `AuthorizationInterceptor` 校验，未携带有效 Token 返回 401
- `**@IgnoreAuth` 白名单**：登录、注册、公开浏览等接口免认证
- **Token 过期机制**：每个 Token 有效期 1 小时，过期后需重新登录

### 5.2 SQL 注入防护

- MyBatis-Plus 使用 `EntityWrapper` 参数化查询
- `SQLFilter` 工具类对排序字段进行关键字黑名单过滤
- 动态 SQL 使用 `${}` 的地方已限制为内部可控参数

### 5.3 跨域配置

- `InterceptorConfig` 配置全局 CORS（`addCorsMappings`）
- `AuthorizationInterceptor` 对 OPTIONS 预检请求直接放行

### 5.4 文件安全

- 上传文件使用时间戳+随机数重命名，防止路径猜测
- 下载接口包含路径穿越防护
- 提供文件加密/解密功能（DES/AES）

## 十、总结与展望

### 10.1 总结

本系统基于 Spring Boot + Vue.js + MySQL 技术栈，实现了一个功能完整的校园技能交易平台。系统涵盖技能发布与浏览、智能推荐匹配、交换申请全流程管理、社区交流、学习资料分享与购买、VIP 会员体系、即时通信等核心功能。通过引入协同过滤推荐算法，有效解决了技能信息过载和匹配效率低的问题。

### 10.2 展望

- **推荐算法优化**：引入更多用户行为特征（如交换评价、在线时长），提升推荐精度
- **移动端适配**：开发小程序或移动端 APP，提高使用便捷性
- **信誉体系完善**：建立更科学的信誉评估模型，引入交换成功率、响应时间等维度
- **支付系统对接**：接入真实支付接口（微信支付/支付宝），替换当前的模拟支付流程
- **安全加固**：引入 JWT 替代自定义 Token、增加接口限流和防刷机制
